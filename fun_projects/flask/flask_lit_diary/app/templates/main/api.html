API Overview

The Literature Diary API provides programmatic access to public notes and allows authenticated users to create, update, and delete their own content.

Base URL

/api/v1

Authentication

  • Public: read-only access to notes
  • Authenticated: create, update, and delete notes
  • Session-based authentication using Flask-Login

Notes Endpoints

GET /api/v1/notes

List notes: anonymous users see only public ones; signed-in users also see their private notes; site admin (id 1) sees all. Each item includes boolean is_public.

POST /api/v1/notes

Create a new note (authentication required). Request body may include optional boolean is_public (default public); the created note and list/detail responses include is_public.

PUT /api/v1/notes/<id>

Update an existing note (owner only); you can change visibility with is_public, reflected in the JSON response.

DELETE /api/v1/notes/<id>

Delete a note (owner only).

id — integer note identifier

Full API description and examples are available on GitHub.
